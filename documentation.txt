Bloque 1: Importar las herramientas y la seguridad
Al inicio del archivo, traemos las herramientas que Python necesita para trabajar. Por defecto, Python es muy básico; con estas líneas le damos "superpoderes".

Python
import os
import discord
from dotenv import load_dotenv
import os: Nos da acceso al sistema operativo de tu laptop. Lo necesitamos para buscar archivos en el disco duro.

import discord: Es la librería principal. Contiene todo el código que sabe cómo hablar con los servidores de Discord (enviar mensajes, leer canales, ver usuarios).

from dotenv import load_dotenv: Es nuestra herramienta de ciberseguridad. Sirve exclusivamente para leer ese archivo secreto llamado .env donde guardas tus tokens, evitando que queden expuestos en el código público.

🔒 Bloque 2: Cargar tus secretos en memoria
Antes de encender el bot, tenemos que decirle dónde buscar la llave de encendido de forma segura.

Python
load_dotenv()
DISCORD_TOKEN = os.getenv("DISCORD_TOKEN")
load_dotenv(): Esta función va y busca el archivo oculto .env en tu carpeta, lee lo que tiene dentro y lo carga en la memoria RAM de tu sistema de forma temporal.

DISCORD_TOKEN = os.getenv("DISCORD_TOKEN"):

DISCORD_TOKEN es una variable (una caja vacía que creamos).

os.getenv(...) va a la memoria de la laptop, busca la etiqueta "DISCORD_TOKEN" que cargamos en el paso anterior, extrae ese texto larguísimo y lo guarda dentro de nuestra caja.

🪪 Bloque 3: Configurar los permisos (Intents)
Discord no deja que cualquier bot haga lo que quiera en un servidor por temas de privacidad. Necesitamos configurar su "credencial de acceso".

Python
intents = discord.Intents.default()
intents.message_content = True
intents = discord.Intents.default(): Creamos una variable llamada intents que contiene el paquete de permisos estándar de Discord (saber quién se une, quién se sale, etc.).

intents.message_content = True: Por privacidad, el contenido de los mensajes de texto viene bloqueado por defecto. Con esta línea, le añadimos a nuestra credencial el permiso explícito de leer lo que la gente escribe en los chats.

🤖 Bloque 4: Crear la instancia del bot
Aquí es donde realmente "nace" nuestro personaje.

Python
client = discord.Client(intents=intents)
client: Es la variable que representa a tu bot activo. Es el objeto con el que haremos todo.

discord.Client(intents=intents): Inicializa el bot y le entrega la credencial de permisos (intents) que configuramos en el bloque anterior para que Discord sepa qué tiene permitido hacer.

⚡ Bloque 5: El evento de conexión (on_ready)
Los bots de Discord funcionan mediante Eventos (cosas que pasan en el servidor a las que el bot reacciona). El primero de ellos es cuando el bot se conecta a internet con éxito.

Python
@client.event
async def on_ready():
    print(f'We have logged in as {client.user}')
@client.event: Es un decorador. Le avisa a Python: "Ojo, la función que viene justo abajo está amarrada a un evento de Discord".

async def: Define una función asíncrona. Los bots deben ser asíncronos porque no pueden congelar todo su código mientras esperan que Discord responda a través de internet; deben poder hacer varias cosas a la vez.

on_ready(): Es un evento con nombre reservado. Discord lo dispara una sola vez en el instante exacto en que tu bot inicia sesión correctamente.

print(...): Imprime un mensaje en tu terminal de Linux (no en el chat de Discord) para avisarte que todo está marchando bien. {client.user} se reemplaza automáticamente con el nombre y número de tu bot.

💬 Bloque 6: El evento de mensajes (on_message)
Este evento se dispara de forma automática cada vez que alguien envía un mensaje en cualquier canal de texto donde esté el bot.

Python
@client.event
async def on_message(message):
message: Es un objeto gigante que Discord nos envía cada vez que alguien escribe. Contiene toda la información del mensaje: quién lo envió (message.author), en qué canal (message.channel), y qué texto tiene (message.content).

🛡️ El sistema de seguridad antibucle:
Python
    if message.author == client.user:
        return
Si tu bot envía un mensaje al canal, este evento on_message se volvería a disparar (porque el bot detecta un mensaje nuevo). Si no pusiéramos este if, el bot leería su propio mensaje, respondería, volvería a leer su respuesta, volvería a responder... y entraría en un bucle infinito que colapsaría el programa.

Esta línea dice: "Si el autor del mensaje (message.author) soy yo mismo (client.user), detén la función inmediatamente (return) y no hagas nada".

🔍 La detección del comando:
Python
    if message.content.startswith('$hello'):
        await message.channel.send('Hello!')
message.content.startswith('$hello'): Revisa si el texto del mensaje empieza exactamente con los caracteres $hello.

await: Le dice a Python: "Enviar este mensaje a través de internet va a tomar unos milisegundos. Pausa esta función aquí y atiende otros procesos del bot mientras Discord nos confirma que el mensaje se envió".

message.channel.send('Hello!'): Toma el canal de texto de donde vino el mensaje original (message.channel) y utiliza la función .send() para escribir 'Hello!' en el chat.

🔑 Bloque 7: El interruptor de encendido
La última línea es la que arranca todo el motor.

Python
client.run(DISCORD_TOKEN)
client.run(...): Toma la variable DISCORD_TOKEN (que tiene tu llave secreta de Discord recuperada del archivo .env) y establece la conexión en tiempo real. A partir de este milisegundo, tu programa se queda escuchando y respondiendo eventos de forma ininterrumpida hasta que cierres la terminal.
